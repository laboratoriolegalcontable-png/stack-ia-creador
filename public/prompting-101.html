El mismo prompt para revisar partes de accidente sueco en 5 versiones. Cada versión añade exactamente un pilar y resuelve el error de la anterior. No intentes escribir v5 de un golpe.

Lección operativa: Cada pilar deja evidencia escrita de por qué está ahí. En producción, si un caso falla, miras el historial de versiones y entiendes qué constraint le faltaba al prompt. Lanza, lee el output, identifica el error específico, añade UN pilar, vuelve a correr.

Técnicas permitidas (solo estas 5)

✅ Role Assignment

Asigna identidad de experto cuando importa el tono o vocabulario técnico. Calibra antes de pedir el output.

✅ Few-Shot Examples (2-5)

Cuando consistencia de formato importa más que la instrucción. 2-5 ejemplos cierran el patrón sin saturar contexto.

✅ XML Structural Tags

Para Claude y prompts complejos con varias secciones. Claude está fine-tuneado para leer bloques XML como intenciones separadas.

✅ Grounding Anchors

Para tareas factuales. Regla tipo “marca [uncertain] si no estás seguro” previene alucinaciones sin saturar el prompt.

✅ Chain of Thought

Para lógica multi-paso, debugging, matemáticas. NUNCA en thinking-native (o3, o4-mini, DeepSeek-R1, Qwen3) — degrada el output.

Técnicas prohibidas (nunca en single-prompt)

❌ Mixture of Experts

El modelo simula personajes en una sola pasada — no hay routing real. Suena bien, fabrica resultados.

❌ Tree of Thought

Genera texto lineal y simula ramificación. No hay paralelismo real, solo apariencia de exploración.

❌ Graph of Thought

Necesita un motor de grafos externo. En single-prompt es fabricación pura.

❌ Universal Self-Consistency

Requiere muestreo independiente. En un solo prompt los caminos posteriores contaminan los anteriores.

❌ Prompt chaining capa-a-capa

En cadenas largas empuja al modelo a fabricar. Usar prompts secuenciales con checkpoints explícitos.

Extended thinking — la muleta del prompt engineer

Extended thinking es herramienta de diagnóstico, no optimización permanente. El flujo:

  1. Activas extended thinking y corres el prompt fallido.
  2. Lees el scratchpad y ves en qué paso Claude se desvía.
  3. Metes esa intuición de regreso al system prompt como regla explícita.
  4. Después de 2-3 iteraciones, apagás thinking — el razonamiento ya vive en el prompt.

No lo prendas en tareas simples — clasificación directa, extracción puntual. Solo mete latencia y costo sin mejorar el output.
